1. Descricao do Projeto

Tema: Sistema de Delivery Local.

Apresentacao do Projeto

Com o crescimento do comercio digital e da utilizacao de servicos de entrega, muitos pequenos estabelecimentos enfrentam dificuldades para gerenciar seus produtos, pedidos e formas de venda de maneira eficiente. Alem disso, a dependencia de grandes plataformas de delivery pode gerar custos elevados devido as taxas cobradas sobre as vendas.

Pensando nesse cenario, este projeto propoe o desenvolvimento de uma API REST para um sistema de delivery local. A aplicacao permitira o cadastro e gerenciamento de usuarios, produtos e formas de venda, oferecendo uma solucao pratica para pequenos negocios que desejam ter maior controle sobre suas operacoes.

Problema que o Sistema Resolve

O sistema busca facilitar a administracao de produtos e servicos disponibilizados por estabelecimentos locais, permitindo que os responsaveis realizem o gerenciamento das informacoes de forma organizada e segura. Dessa maneira, o controle dos dados fica centralizado, reduzindo erros e aumentando a eficiencia operacional.

Publico-Alvo

O sistema foi pensado principalmente para:

Restaurantes;

Lanchonetes;

Pizzarias;

Food trucks;

Pequenos empreendedores do setor alimenticio;

Empresas que trabalham com entregas locais.

5. Levantamento de Requisitos

Requisitos Funcionais

O sistema devera permitir:

Cadastro de usuarios;

Login e autenticacao;

Cadastro de produtos ou servicos;

Consulta de produtos cadastrados;

Atualizacao de informacoes dos produtos;

Exclusao de produtos;

Cadastro de formas de venda;

Atualizacao das formas de venda;

Exclusao das formas de venda;

Geracao de listagens e relatorios;

Associacao dos registros ao usuario responsavel.

Requisitos Nao Funcionais

O sistema devera atender aos seguintes requisitos:

Utilizar autenticacao segura baseada em Token ou JWT;

Ser desenvolvido utilizando Python e Django REST Framework;

Possuir codigo organizado e documentado;

Utilizar Git e GitHub para controle de versoes;

Apresentar desempenho adequado para consultas e operacoes basicas;

Possibilitar futuras expansoes sem grandes alteracoes estruturais.

Para auxiliar na gestao das informacoes, a API disponibilizara endpoints especificos para consultas e relatorios.

Entre eles:

Produtos ativos;

Produtos inativos;

Relatorio resumido contendo o total de produtos cadastrados;

Quantidade de produtos ativos e inativos.

Essas informacoes poderao ser acessadas por meio de requisicoes REST autenticadas.

11. Conclusao

O Sistema de Delivery Local tem como objetivo oferecer uma solucao pratica para o gerenciamento de produtos e formas de venda por meio de uma API REST. O projeto utiliza tecnologias amplamente reconhecidas no mercado, proporcionando seguranca, organizacao e facilidade de manutencao.

Alem de atender aos requisitos propostos na disciplina, a aplicacao podera servir como base para futuras expansoes, incluindo integracao com aplicativos moveis, sistemas de pedidos e plataformas de pagamento online.
